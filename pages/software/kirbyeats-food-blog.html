Project Description

Kirby Eats is a web platform for food sharing that lets food enthusiasts discover, share, and review the best restaurants around the world. Users are able to generate posts to illustrate a range of cuisines, evaluate their experiences, present location details, and include pictures. This application has the feature to separate posts by cuisine, adjust price filters, handle dietary preferences, and sort results according to rating or popularity. Kirby Eats has become an energetic and interactive tool for food discovery and interaction due to the fact that users have the opportunity to join groups, track those passionate about food, and express their insights via comments.

There are two implementations for this project: one using MySQL and the other using NoSQL (MongoDB). These implementations cater to different database paradigms, emphasizing structured relational data versus flexible document-based storage.

‚öñÔ∏è Main Differences Between MySQL and NoSQL Implementations

  1. Database Design:
    • MySQL: Uses normalized tables with strict schemas for entities such as Users, Posts, Comments, and Communities. Joins are utilized to establish relationships between entities.
    • NoSQL: Stores data in collections with flexible schemas, enabling quicker iteration. Embedded documents and aggregation pipelines replace traditional joins.
  2. Query Handling:
    • MySQL: Relies on SQL queries with table joins, indexes, and transactions for consistent, reliable data retrieval.
    • NoSQL: Employs aggregation pipelines and embedded sub-documents, reducing query complexity in exchange for potentially higher data duplication.
  3. Scalability:
    • MySQL: Best suited for vertical scaling (adding resources to a single server).
    • NoSQL: Optimized for horizontal scaling (adding more servers), accommodating large-scale, unstructured datasets.
  4. Performance:
    • MySQL: Ideal for structured data and transactional consistency.
    • NoSQL: Excels in handling high read and write loads with flexible schema modifications.
