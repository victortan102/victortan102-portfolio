Project Information

  • Category: Hardware, Embedded Systems Programming
  • Language: C
  • Project Date: October 2019 - February 2020

Project Introduction and Objectives

Mr Tan owns a company that manufactures domestic electric bathroom water heaters. They use a thermostat as the main automatic temperature control device. This kind of water heater is becoming outdated in the market.

We have been approached by Mr Tan to produce a prototype for a smart environment and user-friendly water heater with useful features to set water flow rate and heating time. The prototype shall eventually provide useful attributes to the management team to decide on the feasibility of manufacturing such water heaters.

Our team has decided to add the following features to the prototype to produce a smart bathroom water heater for the company.

  • The heater's water level can be set by turning the trimmer to the user desired flow rate (0.0 litres/min to 8.9 litres/min).
  • The water heater's heat level can be set by pressing a number on the keypad to change the level of heat from 0 to 9.

System Operation (How the program works)

Our project, the smart water heater, is used as a prototype to showcase various user-friendly features for controlling the water heater. The user will be able to control the water heater using four different components:

  • Keypad (RB0)
  • S3 (RB2)
  • Trimmer (RA0)
  • Touch Sensor (RB1)

The user can adjust the water heater’s heating level by pressing values 0 to 9 on the keypad with value 0 equals no heating, whereas value 9 is the highest heating level.

However, if the user presses an invalid key, the LCD will display an error message and prompt the user to re-enter a valid input.

When the heating level selected is of a higher temperature than the current temperature, the light bulb will increase its brightness with the help of the Timer1 Module in the PIC18F4520 microcontroller counting at a speed of 200ms as it increases a preset count value until it reaches the desired temperature.

On the other hand, when the temperature is lower than the current temperature, the light bulb’s brightness will decrease accordingly to the new preset count value of the Timer1 Module in the PIC18F4520 microcontroller until it reaches the new desired temperature.

Secondly, the water flow rate can be adjusted when the push button ‘S3’ is pressed to enable the user to change the flow rate. Thereafter, the user can adjust the water flow rate by turning the trimmer on the water heater. The flow rate will be displayed on the 7-segment display with 0.0 litres/min, indicating that no water is flowing to 8.9 litres/min, indicating that the water heater is running at the maximum flow rate. An audible sound will also be available to allow the user to hear.

Next, the touch sensor will be used to simulate a motion sensor to detect if the user is present. If the sensor does not sense any user after 60 seconds, the system will go into sleep mode. Whereas, if the sensor detects the user, the system will continue execution until there is no user interaction.

The SLEEP mode of the microcontroller will be activated after 60 secs of inactiveness. In order to provide a visual representation of the microcontroller’s current power mode, the LED colour will change from green to yellow to red through an interval of 20 seconds for each LED:

  • At 20 secs, the yellow LED will light up to warn the user that the system has gone past 20 secs without user interaction.
  • After 40 secs, the red LED will light up, indicating that the system has gone past 40 secs without user interaction.
  • After another 20 secs, the system will enter into the SLEEP Mode. On the sleep instruction, the bulb will also be turned off to conserve energy.

To wake up the system, the user can touch the touch sensor or use any of the controls such as the keypad or switch S3 to continue using the system.
